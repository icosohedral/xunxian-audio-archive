# 冷调数字档案设计系统

版本：1.0
用途：供 AI 在网站设计、UI 生成、前端开发和现有界面改造时使用。

## 1. AI 的任务

你需要使用本设计系统，为当前项目创建一套冷静、克制、内容优先的数字界面。

界面应呈现以下气质：

* 数字档案
* 研究工具
* 仪器面板
* 编辑记录
* 历史材料
* 数据索引
* 旧式数字媒介

设计的重点不是制造“高级感”或“科技感”，而是让信息像材料、记录或证据一样被阅读。

界面必须减少装饰，将视觉层级主要建立在字体、字号、明度、细线、留白和信息位置上。

不要机械复制时间轴、岩芯、社区颜色或其他原项目特有元素。应根据新项目的内容结构，重新寻找适合的核心视觉隐喻。

---

## 2. 核心设计原则

### 2.1 内容优先

内容是界面的主体，界面是用于测量、分类、定位和解释内容的工具。

不要让容器比内容更显眼。不要用大量卡片、渐变、阴影或高饱和颜色制造层级。

优先通过以下方式区分信息：

1. 字体角色
2. 字号
3. 字重
4. 文字明度
5. 留白
6. 细分隔线
7. 少量语义色

### 2.2 冷静而非冷漠

整体应保持理性、安静、低刺激，但不能降低可读性。

使用带轻微蓝色倾向的冷灰，而不是纯白和纯黑。正文对比度必须足够，辅助信息可以弱化，但不能模糊。

### 2.3 平面而非悬浮

常规内容应直接排列在页面平面上。

除弹窗、抽屉、浮动菜单等真实浮层外，不要给普通模块添加阴影。

不要把每一段内容都包裹成独立圆角卡片。优先使用细线、间距和排版形成分组。

### 2.4 颜色必须具有含义

颜色只用于表达：

* 内容类别
* 来源
* 状态
* 数据系列
* 当前选中项
* 搜索命中
* 需要注意的信息

不要仅为了装饰而加入颜色。

同一颜色在一个项目中必须保持稳定含义。不要让两个需要被快速区分的类别使用过于接近的颜色。

### 2.5 允许留白表达信息

空白不只是装饰。

当内容中存在时间间隔、缺失数据、沉默期、阶段切换或未发生事件时，可以让空间本身表达这种差异，而不是用额外图标或说明填满页面。

## 3. 配色系统

### 3.1 基础配色

默认使用冷灰蓝色系。

```css
:root {
  --color-bg: #e9edf1;
  --color-surface: #f3f6f9;

  --color-text: #131a24;
  --color-text-secondary: #55606d;
  --color-text-muted: #8d97a3;

  --color-border: #d2dae2;
  --color-highlight: #e0a93c;

  --color-accent-1: #2c5aa8;
  --color-accent-2: #2e8b45;
  --color-accent-3: #b0691b;
}
```

颜色角色：

* `--color-bg`：页面主背景
* `--color-surface`：输入框、浮层按钮或轻微区分区域
* `--color-text`：正文和重要标题
* `--color-text-secondary`：次级正文和补充信息
* `--color-text-muted`：时间、数量、标签、字段名
* `--color-border`：分隔线和控件边框
* `--color-highlight`：搜索命中或临时强调
* `--color-accent-*`：分类、状态或数据系列

### 3.2 配色比例

建议视觉占比：

* 中性色：85%–95%
* 分类或品牌色：5%–12%
* 高亮色：少于 3%

禁止大面积使用高饱和品牌色作为常规模块背景。

### 3.3 深色模式

深色模式必须通过语义变量切换，不要为每个组件单独维护一套颜色。

```css
:root[data-theme="dark"] {
  --color-bg: #0f141a;
  --color-surface: #161d26;

  --color-text: #d7dee6;
  --color-text-secondary: #8a95a2;
  --color-text-muted: #59636f;

  --color-border: #242d38;
  --color-highlight: #e9b84d;

  --color-accent-1: #6e9be8;
  --color-accent-2: #5fbf7a;
  --color-accent-3: #d9a05b;
}
```

深色模式不是简单反相。

需要分别调整：

* 背景层级
* 文字明度
* 边框可见度
* 语义色亮度
* hover 叠加方向

主题值应在页面首次绘制前确定，避免加载时出现浅色闪烁。

## 4. 字体系统

使用三种字体角色，而不是只使用一种全局字体。

```css
:root {
  --font-content:
    "Songti SC",
    "Songti TC",
    "SimSun",
    "宋体",
    serif;

  --font-ui:
    "PingFang SC",
    "Hiragino Sans GB",
    "Microsoft YaHei",
    system-ui,
    sans-serif;

  --font-data:
    "SF Mono",
    "Menlo",
    "Consolas",
    ui-monospace,
    monospace;
}
```

### 4.1 内容字体

衬线体用于具有作者声音、历史属性或长篇阅读性质的内容，例如：

* 文章正文
* 用户原话
* 引用
* 档案记录
* 访谈
* 日记
* 信件
* 作品标题

内容字体的作用是让原始材料与现代界面框架产生区别。

### 4.2 界面字体

无衬线字体用于：

* 导航
* 按钮
* 表单
* 筛选器
* 菜单
* 提示
* 上下文说明
* 空状态
* 系统信息

界面字体应保持中性，不模仿内容本身的语气。

### 4.3 数据字体

等宽字体用于：

* 时间
* 日期
* 年份
* 数量
* 百分比
* 编号
* 状态代码
* 类型标签
* 数据字段
* 版本号
* 统计图表

数字应优先启用等宽数字：

```css
font-variant-numeric: tabular-nums;
```

### 4.4 建议字号

```css
--font-size-xs: 10px;
--font-size-sm: 12px;
--font-size-ui: 13px;
--font-size-body-sm: 15px;
--font-size-body: 17px;
--font-size-title: 21px;
--font-size-display: 30px;
```

使用建议：

* 数据标签：9–10.5px
* 界面辅助信息：12px
* 常规控件：12–13px
* 次级正文：15px
* 主要正文：16–18px
* 页面标题：20–24px
* 核心年份或关键数据：26–34px

正文建议行高为 `1.6–1.7`。
界面文字建议行高为 `1.45–1.6`。

不要通过过重字重制造层级。大字号标题可以只使用 `500` 或 `600`。

## 5. 排版规则

### 5.1 信息层级

页面层级应尽量控制在以下结构内：

1. 页面标题
2. 章节或阶段标题
3. 内容标题
4. 正文
5. 上下文信息
6. 数据和元信息

每一级必须至少通过字号、字体角色、明度或间距中的两项产生区别。

### 5.2 小型标签

字段名、数据区标题和分类标题使用等宽字体、小字号、大字间距：

```css
.data-label {
  font-family: var(--font-data);
  font-size: 10px;
  font-weight: 400;
  letter-spacing: 0.12em;
  color: var(--color-text-muted);
}
```

标签应像仪器读数或档案字段，而不是营销式副标题。

### 5.3 文本宽度

长篇阅读区域建议最大宽度为 `680–800px`。

不要让正文跨越整个宽屏。宽屏空间可用于筛选器、目录、数据概览或辅助导航。

### 5.4 长文本

长文本可在列表状态下折叠，但必须：

* 仅在确实溢出时显示展开按钮
* 保持完整文本可访问
* 使用渐隐暗示剩余内容
* 展开后避免页面横向跳动
* 不使用固定像素高度模拟行数

建议：

```css
.clamped {
  max-height: calc(1.65em * 5);
  overflow: hidden;
}
```

## 6. 布局系统

### 6.1 主内容布局

优先使用单列、线性或文档式布局。

适合的结构包括：

* 时间线
* 版本记录
* 文章流
* 档案列表
* 事件日志
* 更新记录
* 搜索结果
* 研究笔记
* 数据叙事

不要默认使用三列卡片网格。

### 6.2 辅助区域

桌面端可将以下内容放置在主内容两侧：

* 筛选器
* 目录
* 时间索引
* 数据概览
* 分类图例
* 状态摘要

辅助区域不能压缩主内容到难以阅读。

当空间不足时，辅助区域应转换为：

* 抽屉
* 弹层
* 折叠区域
* 顶部筛选入口

### 6.3 几何变量

具有共同对齐关系的组件必须共享几何变量，不要在多个选择器中重复魔法数字。

```css
.timeline {
  --axis-x: 62px;
  --content-offset: 88px;
  --page-padding: 20px;
  --node-size: 8px;
}
```

轴线、节点、标题和内容必须使用同一组变量计算位置。

---

## 7. 内容容器

普通内容项不要设计成悬浮卡片。

推荐样式：

```css
.content-item {
  position: relative;
  padding: 13px 4px 14px 0;
  border-bottom: 1px solid var(--color-border);
}

.content-item:hover {
  background:
    color-mix(
      in srgb,
      var(--color-text) 4%,
      transparent
    );
}
```

内容项可以拥有：

* 左侧节点
* 顶部上下文
* 标题
* 正文
* 底部元信息
* 类型标签

内容项不应默认拥有：

* 大圆角
* 明显阴影
* 白色独立背景
* 粗边框
* 大面积品牌色
* hover 上浮
* 夸张缩放

只有真正可拖拽或空间层级发生变化的元素，才可以使用上浮动效。

## 8. 控件设计

### 8.1 通用控件

```css
.control {
  min-height: 28px;
  padding: 3px 10px;

  border: 1px solid var(--color-border);
  border-radius: 2px;

  background: transparent;
  color: var(--color-text-secondary);

  font-family: var(--font-ui);
  font-size: 12px;
}
```

控件应保持：

* 细边框
* 小圆角
* 较低高度
* 透明或接近背景色的底色
* 明确的 focus 状态

### 8.2 按钮层级

默认按钮不要使用实色背景。

只有页面中唯一或明确的主要操作可以使用实色强调。主要操作数量应受到严格限制。

危险操作不要只依赖红色，还必须通过文字、图标或确认步骤表达风险。

### 8.3 筛选项

筛选项可以用一个小色点表达分类。

选中状态通过以下组合表示：

* 文字颜色
* 字重
* 色点
* 必要时增加浅色背景

不要仅依赖背景填充表达选中状态。

长名称必须换行，不要使用省略号隐藏关键分类信息。

---

## 9. 边框、圆角与阴影

### 9.1 边框

默认边框为 `1px`。

边框主要用于：

* 内容分隔
* 控件边界
* 弹窗边界
* 数据区分
* 当前选择轮廓

### 9.2 圆角

建议范围：

* 标签：1–2px
* 按钮和输入框：2px
* 弹窗：3–4px
* 圆形节点：仅用于具有语义的标记

避免使用超过 `8px` 的常规圆角。

不要把大圆角当作“现代感”的默认来源。

### 9.3 阴影

阴影仅用于真实浮层：

* 抽屉
* 弹窗
* 下拉菜单
* 浮动工具条

普通内容项和静态面板不使用阴影。

---

## 10. 数据可视化

数据图形应保持平面、直接和低装饰。

优先使用：

* 细条形图
* 线性刻度
* 色带
* 小型趋势线
* 数量比例
* 紧凑型数据云
* 带语义的节点形状

数据视觉编码应遵循：

* 颜色表达类别
* 长度或厚度表达数量
* 位置表达时间或顺序
* 形状表达来源或类型

不要让颜色、形状和大小同时表达同一个变量。

每种视觉编码必须有稳定含义。

---

## 11. 动效系统

动效只用于反馈状态，不用于持续装饰。

建议时长：

```css
--duration-fast: 120ms;
--duration-normal: 160ms;
--duration-panel: 220ms;
--duration-reveal: 250ms;
```

允许的动效：

* 颜色渐变
* 边框变化
* 淡入淡出
* 抽屉滑入
* 弹窗轻微上移
* 定位目标短暂高亮
* 返回顶部按钮出现

避免：

* 弹跳
* 大幅缩放
* 持续呼吸
* 大面积视差
* 自动循环动画
* 内容卡片上浮
* 与任务无关的背景动画

弹窗位移建议控制在 `6–10px`，缩放不低于 `0.98`。

必须支持减少动态效果：

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation: none !important;
    transition: none !important;
    scroll-behavior: auto !important;
  }
}
```

## 12. 响应式规则

响应式设计的目标不是缩小桌面页面，而是重新分配信息优先级。

### 宽屏

* 主内容保持有限宽度
* 左侧可显示筛选器或目录
* 右侧可显示辅助数据索引
* 顶部工具栏与内容建立对齐基线

### 中等屏幕

* 常驻侧栏改为抽屉
* 主内容居中
* 保留完整正文
* 减少非核心辅助图形

### 小屏幕

* 减少左右缩进
* 将轴线移向屏幕边缘
* 正文保持至少 `16px`
* 隐藏非必要概览图
* 长名称允许换行
* 交互目标不得过小
* 弹窗宽度使用视口减固定边距

不要通过明显缩小正文解决空间问题。

## 13. 可访问性

必须实现：

```css
:focus-visible {
  outline: 2px solid var(--color-text);
  outline-offset: 2px;
}
```

同时遵守：

* 所有交互元素可通过键盘使用
* 不仅依赖颜色表达状态
* 正文和背景保持足够对比
* 弱化文字仍需可读
* 图标按钮提供可访问名称
* 弹窗打开后管理焦点
* 抽屉打开时阻止背景误操作
* 图片和图表提供文本说明
* 搜索高亮不能遮盖文字
* hover 信息必须也能通过 focus 获得

固定顶栏存在时，滚动定位必须考虑顶栏高度。

## 14. 项目适配方法

开始新项目时，AI 必须先识别以下内容：

1. 项目的主要内容是什么
2. 哪些内容属于原始材料
3. 哪些内容属于现代界面
4. 哪些内容属于数据或元信息
5. 项目中有哪些稳定分类
6. 用户最常进行的操作是什么
7. 哪些辅助信息可以在小屏幕隐藏
8. 是否存在适合作为视觉主线的结构

然后生成项目专属设计：

* 为原始材料选择内容字体
* 为界面选择无衬线字体
* 为数据选择等宽字体
* 为每个稳定分类指定语义色
* 确定主内容最大宽度
* 确定桌面辅助面板的位置
* 确定移动端的信息折叠方式
* 设计一个与项目内容有关的签名元素

签名元素必须来源于项目自身，而不是为了独特而独特。

可能的签名元素：

* 时间沉积层
* 编辑痕迹
* 页码索引
* 文件标签
* 版本分支
* 事件密度
* 地理路径
* 声音波形
* 阅读进度
* 资料来源结构

---

## 15. 禁止项

除非项目明确要求，否则不要使用：

* 大面积纯白背景
* 纯黑正文
* 大面积高饱和渐变
* 玻璃拟态
* 新拟态
* 霓虹发光
* 大圆角卡片
* 卡片瀑布流
* 每个模块都有阴影
* 夸张的英雄区域
* 巨型营销标题
* 装饰性 3D 图形
* 无意义的插画
* 持续循环动画
* 为所有按钮添加品牌色
* 用颜色代替信息层级
* 用省略号隐藏重要名称
* 为移动端过度缩小文字
* 无条件添加毛玻璃模糊
* 用动画掩盖加载或布局问题

## 16. AI 输出要求

使用本规范设计项目时，输出至少包括：

1. 设计概念
   用一段话解释界面与项目内容之间的关系。

2. 语义色变量
   提供浅色和深色主题。

3. 字体角色
   明确内容字体、界面字体和数据字体。

4. 排版尺度
   包含正文、标题、标签和元信息。

5. 布局说明
   解释桌面端和移动端结构。

6. 核心组件
   包含导航、内容项、筛选器、按钮、输入框、浮层和空状态。

7. 交互状态
   包含默认、hover、focus、active、disabled 和 loading。

8. 响应式规则
   说明哪些结构重排、折叠或隐藏。

9. 可访问性实现
   包含键盘操作、焦点和减少动态效果。

10. CSS 变量
    所有重复颜色、字体、间距和几何尺寸都应变量化。

生成代码时，应优先使用：

* 语义化 HTML
* CSS 自定义属性
* Grid 和 Flexbox
* `color-mix()`
* `clamp()`
* `min()`
* `max()`
* `calc()`
* `prefers-color-scheme`
* `prefers-reduced-motion`
* `focus-visible`

不要引入不必要的 UI 框架或动画库。

## 17. 验收标准

最终页面应满足以下判断：

* 去掉品牌色后，层级仍然清晰。
* 去掉阴影后，结构仍然成立。
* 页面看起来像内容工具，而不是营销落地页。
* 用户能快速区分内容、界面和数据。
* 大多数区域依靠排版和细线组织，而不是卡片。
* 颜色数量有限且含义稳定。
* 长文本适合持续阅读。
* 深色模式不是简单反相。
